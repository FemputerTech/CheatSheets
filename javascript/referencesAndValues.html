References and Values

Primitive Data Types (Values)

Such as: string, number, boolean, etc.

These hold their actual value. When you assign a primitive value to a variable or pass it to a function, a copy of that value is created

Key characteristics:

  • Pass-by-value: When you assign a primitive to a variable, or pass it to a function, you get a copy of the value.
  • Immutability: You can't modify the original primitive value directly once it's been set. However, you can assign a new value to the variable.
  • Reassigning: Reassigning a primitive means setting it to a new value.

Example

        let a = 10;
        let b = a; // Copy of the value, not a reference

        b = 20; // Reassigning b won't affect a
        console.log(a); // 10
        console.log(b); // 20

Objects and Arrays (References)

Objects (including arrays and functions) in JavaScript are reference types. When you assign an object to a variable or pass it to a function, you're not copying the actual object. Instead, you're copying the reference (or memory address) where the object is stored.

Key characteristics:

  • Pass-by-refernce: When you assign an object to a variable, or pass it to a function, you are passing the reference to the original object, not a copy of the object.
  • Mutability: Objects can be modified. If you change a property of an object or an array, those changes affect all references to the object.
  • Reassigning: Reassigning a reference means pointing it to a different memory location.

Example

        let obj1 = { name: 'Alice' };
        let obj2 = obj1; // obj2 is a reference to the same object as obj1

        obj2.name = 'Bob'; // Modifying obj2 also affects obj1
        console.log(obj1.name); // Bob
        console.log(obj2.name); // Bob
