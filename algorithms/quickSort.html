QUICK SORT

Use Cases

  • Large datasets
  • Systems with limited memory

Steps

  1. Choose a "pivot" element (i.e. first element)
  2. Put everything smaller than pivot into left array and everything greater into right array
  3. Recursively apply the same process to left and right
  4. Base case when subarray has < 2 elements
  5. Continue until sorted

Time Complexity

  • Worst Case: O(nÂ²) (poorly chosen pivot)
  • Average Case: O(nlogn)
  • Best Case: O(nlogn)

Space Complexity

  • O(logn) (due to recursion)

Advantages

  • Very efficient on average
  • Performs well for large datasets
  • In-place sorting (small amount of additional memory)

Disadvantages

  • Poor pivot choice causes imbalanced partitions
  • Unstable: relative order may not be preserved
  • Recursion can lead to stack overflow
