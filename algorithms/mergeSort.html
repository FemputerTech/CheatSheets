divide and conquer

MERGE SORT

Use Cases

  • Large datasets
  • Linked lists
  • Require stable sorting

Steps

  1. Divide: Split until each subarray contains 1 element (base case).
  2. Conquer: Recursively sort
  3. Combine: Merge sorted halves back together:
    • Compare elements of each half
    • Insert smaller into merged array
    • Continue until all elements are merged

Time Complexity

  • O(nlogn)

Space Complexity

  • O(n) (temporary arrays)

Advantages

  • Efficient for large datasets and linked lists
  • Performs consistently well
  • Stable: preserves relative order

Disadvantages

  • Less memory efficient
  • Slower for smaller datasets due to recursive call overhead
