INSERTION SORT

Use Cases

  • Small datasets
  • Partially sorted arrays
  • Data comes in small increments

Steps

  1. Consider first element as sorted & start with second
  2. Compare current element with elements in sorted portion
  3. Shift all elements larger than current one position to the right
  4. Move to next and repeat until sorted

Time Complexity

  • Worst Case: O(n²)
  • Average Case: O(n²)
  • Best Case: O(n) (already nearly sorted)

Space Complexity

  • O(1) (in-place)

Advantages

  • Efficient for small or nearly sorted datasets
  • Simple to implement
  • Stable: preserves relative order

Disadvantages

  • Inefficient for large datasets
  • O(n²) in avg and worst cases
